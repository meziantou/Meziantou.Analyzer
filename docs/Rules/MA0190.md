# MA0190 - Use partial property instead of partial method for GeneratedRegex

C# 13 introduced support for partial properties. When using `[GeneratedRegex]`, a partial property is preferred over a partial method because it provides a more natural syntax: accessing the regex is a simple property access rather than a method call.

This rule reports `[GeneratedRegex]` partial methods and suggests converting them to partial properties. The code fix also updates all call sites by removing the empty argument list `()`.

## Non-compliant code

```csharp
[GeneratedRegex(@"sample.*", RegexOptions.CultureInvariant, matchTimeoutMilliseconds: Timeout.Infinite)]
private static partial Regex SampleRegex();

// Usage
SampleRegex().Match("");
```

## Compliant code

```csharp
[GeneratedRegex(@"sample.*", RegexOptions.CultureInvariant, matchTimeoutMilliseconds: Timeout.Infinite)]
private static partial Regex SampleRegex { get; }

// Usage
SampleRegex.Match("");
```
