# MA0183 - string.Format should use a format string with placeholders

## Description

This rule detects calls to `string.Format` where the format string is a constant that contains no format placeholders. This is inefficient and often indicates a mistake, as the parameters passed to `string.Format` will not be used.

## Violations

```csharp
// No parameters provided
string result = string.Format("value without argument");

// Parameters provided but format string has no placeholders
string result = string.Format("value with argument", 123);

// Parameters provided but format string only has escaped braces
string result = string.Format("value with argument {{0}}", 123);

// With IFormatProvider but no placeholders
string result = string.Format(CultureInfo.InvariantCulture, "no placeholder", 123);
```

## Valid Code

```csharp
// Valid: format string has placeholders
string result = string.Format("value with argument {0}", 123);

// Valid: format string is not a constant (can't be analyzed)
string format = "test {0}";
string result = string.Format(format, 123);

// Valid: escaped braces with placeholders
string result = string.Format("value {{escaped}} {0}", 123);
```

## How to Fix Violations

Replace `string.Format` with a string literal or string interpolation when no placeholders are used:

```csharp
// Before
string result = string.Format("value without argument");

// After
string result = "value without argument";
```

If parameters are passed but not used, ensure you add the appropriate placeholders:

```csharp
// Before
string result = string.Format("value with argument", 123);

// After
string result = string.Format("value with argument {0}", 123);
// Or use string interpolation
string result = $"value with argument {123}";
```
