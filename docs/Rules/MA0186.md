# MA0186 - Use String.Create with IFormatProvider when all interpolated string parameters use the same format provider

## Description

When all parameters of an interpolated string use the same format provider (IFormatProvider) to convert values to strings, it's more efficient to use `String.Create(IFormatProvider?, ref DefaultInterpolatedStringHandler)` instead. This method allows you to specify the format provider once rather than passing it to each individual ToString call.

## Problematic Code

```csharp
// All ToString calls use the same format provider
var x = $"{DateTime.Now.ToString("D", CultureInfo.InvariantCulture)}{DateTime.Now.ToString("D", CultureInfo.InvariantCulture)}";
```

## Solution

```csharp
// Use String.Create with the common format provider
var x = string.Create(CultureInfo.InvariantCulture, $"{DateTime.Now:D}{DateTime.Now:D}");
```

## Additional Information

This rule only reports a diagnostic when:
- There are at least 2 interpolation expressions that call ToString with an IFormatProvider
- All these ToString calls use the same format provider (e.g., all use CultureInfo.InvariantCulture)

The rule will not report a diagnostic when:
- Only one ToString call uses a format provider
- Different format providers are used
- The interpolated string is already used with String.Create
- The interpolated string is converted to FormattableString
