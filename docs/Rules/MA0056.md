# MA0056 - Do not call overridable members in constructor
<!-- sources -->
Source: [DoNotCallVirtualMethodInConstructorAnalyzer.cs](https://github.com/meziantou/Meziantou.Analyzer/blob/main/src/Meziantou.Analyzer/Rules/DoNotCallVirtualMethodInConstructorAnalyzer.cs)
<!-- sources -->

When a virtual method or property is called, the actual type that executes the member is not selected until run time. When a constructor calls a virtual method or property, it's possible that the constructor for the actual instance type has not been executed yet.

````csharp
class Sample
{
    public Sample()
    {
        // What does it do?
        // new Sample2() prints "0"
        VirtualMethod(); // Non compliant
        _ = VirtualProperty; // Non compliant
    }

    protected virtual void VirtualMethod()
    {
    }

    protected virtual int VirtualProperty => 0;
}

class Sample2 : Sample
{
    private int value;

    public Sample2() => value = 42;

    protected override void VirtualMethod()
    {
        Console.WriteLine(value);
    }

    protected override int VirtualProperty => value;
}
````

The diagnostic is not reported in `sealed` classes.

Note: Unlike [CA2214](https://learn.microsoft.com/en-us/dotnet/fundamentals/code-analysis/quality-rules/ca2214?WT.mc_id=DT-MVP-5003978), MA0056 also reports:
- Virtual property accesses
- Calls to virtual members inherited from a base class
