# MA0181 - Do not use cast

<!-- sources -->
Source: [DoNotUseCastAnalyzer.cs](https://github.com/meziantou/Meziantou.Analyzer/blob/main/src/Meziantou.Analyzer/Rules/DoNotUseCastAnalyzer.cs)
<!-- sources -->

This rule is disabled by default as casts are sometimes necessary and commonly used. However, if you want to enforce a stricter type-safe coding style, you can enable this rule.

**Note**: This rule only reports explicit casts, not implicit conversions, `as` operators, or `is` pattern matching.

````c#
int value = 42;
double result = (double)value; // non-compliant

object obj = "test";
string str = (string)obj; // non-compliant

MyEnum enumValue = MyEnum.Value1;
int intValue = (int)enumValue; // non-compliant

// Compliant alternatives:
double result2 = value; // implicit conversion (not reported)
string? str2 = obj as string; // as operator (not reported)
if (obj is string str3) { } // is pattern (not reported)
````
