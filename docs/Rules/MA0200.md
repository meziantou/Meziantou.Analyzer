# MA0200 - Do not use empty property patterns with non-nullable value types
<!-- sources -->
Sources: [DoNotUseEmptyPropertyPatternOnNonNullableValueTypeAnalyzer.cs](https://github.com/meziantou/Meziantou.Analyzer/blob/main/src/Meziantou.Analyzer/Rules/DoNotUseEmptyPropertyPatternOnNonNullableValueTypeAnalyzer.cs), [DoNotUseEmptyPropertyPatternOnNonNullableValueTypeFixer.cs](https://github.com/meziantou/Meziantou.Analyzer/blob/main/src/Meziantou.Analyzer.CodeFixers/Rules/DoNotUseEmptyPropertyPatternOnNonNullableValueTypeFixer.cs)

`is { }` checks whether a value is non-null.
For non-nullable value types, this check is always `true`.
Similarly, `is not { }` is always `false`.

````c#
int value = 0;

_ = value is { };      // non-compliant
_ = value is not { };  // non-compliant
````

````c#
int? value = 0;

_ = value is { };      // compliant
_ = value is not { };  // compliant
````
