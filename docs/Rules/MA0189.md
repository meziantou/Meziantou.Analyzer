# MA0189 - Use InlineArray instead of fixed-size buffers

C# 12 introduced `InlineArrayAttribute`, which provides a safer and more modern pattern than C-style fixed-size buffers.

This rule reports fields declared with the `fixed` keyword such as `public fixed byte Data[16];` and suggests replacing them with an inline array type.

When `System.Runtime.CompilerServices.InlineArrayN<T>` (for `N` from 2 to 16) is available, a code fix is provided to replace the fixed-size buffer with the matching `InlineArrayN<T>` field type.

## Non-compliant code

```csharp
unsafe struct Packet
{
    public fixed byte IpAddress[16];
}
```

## Compliant code

```csharp
using System.Runtime.CompilerServices;

[InlineArray(16)]
public struct Byte16
{
    private byte _element0;
}

public struct Packet
{
    public Byte16 IpAddress;
}
```
